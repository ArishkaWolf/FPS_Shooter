# FPS Shooter — Симуляция распространения звука в Unreal Engine 5

## О проекте

Данный проект разработан в рамках выпускной квалификационной работы на тему:

**«Симуляция распространения звука в 3D-играх от первого лица»**

Проект представляет собой исследовательскую систему моделирования распространения звука в игровой среде Unreal Engine 5 на основе трассировки звуковых лучей.

В отличие от традиционной радиусной модели слуха, звук рассматривается как совокупность лучей, которые распространяются по сцене, взаимодействуют с препятствиями, отражаются от поверхностей, теряют энергию на расстоянии и могут быть восприняты неигровыми персонажами.

Основной целью проекта является исследование влияния количества звуковых лучей на качество моделирования распространения звука и работу системы слуха NPC.

---

## Основные возможности

### Источник звука

В качестве демонстрационного источника используется падающая сфера.

После столкновения с поверхностью происходит генерация звукового события.

Для источника настраиваются:

* громкость;
* количество лучей;
* коэффициент затухания;
* коэффициент поглощения стен;
* максимальное количество отражений.

---

### Система распространения звука

Реализована модель распространения звука на основе трассировки лучей.

Поддерживаются:

* ненаправленный источник звука;
* генерация набора лучей;
* отражение от препятствий;
* поглощение звука поверхностями;
* затухание на расстоянии;
* визуализация лучей в сцене.

---

### Система слуха NPC

Каждый NPC обладает собственным компонентом слуха.

Система позволяет:

* определять достижение звука персонажа;
* вычислять силу полученного сигнала;
* хранить последнюю услышанную позицию;
* отображать процент полученного звука;
* постепенно забывать звуковое событие.

После получения сигнала NPC поворачивается в сторону последнего услышанного звука.

---

### Автоматический демонстрационный полигон

При запуске проекта автоматически создаётся тестовая сцена.

Полигон включает:

* источник звука;
* набор препятствий;
* пять NPC;
* визуализацию распространения звуковых лучей.

Это позволяет проводить повторяемые эксперименты без ручной настройки уровня.

## Архитектура системы

Проект построен на компонентной архитектуре Unreal Engine 5.

Основные классы:

### AFallingSoundSphere

Источник звука.

Отвечает за:

* физическую симуляцию;
* столкновения;
* создание звукового события.

### USoundRayEmitterComponent

Основной компонент распространения звука.

Отвечает за:

* генерацию лучей;
* трассировку лучей;
* отражения;
* расчёт затухания;
* передачу сигнала NPC.

### UNPCHearingComponent

Компонент слуха NPC.

Отвечает за:

* приём звукового сигнала;
* хранение силы сигнала;
* хранение последней услышанной позиции;
* забывание звука.

### ASoundHearingNPC

Неигровой персонаж.

Отображает:

* уровень полученного сигнала;
* реакцию на звук.

### USoundRayDemoSubsystem

Подсистема автоматического создания тестового полигона.

Создаёт:

* источник звука;
* препятствия;
* NPC.

## Математическая модель

### Затухание звука

Громкость уменьшается по мере прохождения расстояния:

V_end = V_start − L · A

где:

* V_start — громкость в начале участка;
* V_end — громкость в конце участка;
* L — расстояние в метрах;
* A — коэффициент затухания.

В проекте используется:

AttenuationPerMeter = 0.04

---

### Поглощение звука препятствиями

После столкновения со стеной часть энергии теряется:

V_reflected = V_end · (1 − W)

где:

* W — коэффициент поглощения стены.

В проекте используется:

WallAbsorption = 0.35

Это означает, что стена поглощает 35% энергии звука, а отражает 65%.

---

### Отражение лучей

Направление отражённого луча рассчитывается по формуле:

R = D − 2(D · N)N

где:

* D — направление падающего луча;
* N — нормаль поверхности;
* R — направление отражённого луча.

## Исследование количества лучей

В ходе работы были исследованы три конфигурации системы:

### 8 лучей

* шаг между лучами 45°
* минимальная вычислительная нагрузка
* низкая плотность покрытия пространства

### 16 лучей

* шаг между лучами 22,5°
* более равномерное покрытие
* улучшенное обнаружение NPC

### 25 лучей

* шаг между лучами 14,4°
* наиболее плотное покрытие пространства
* наиболее качественное распространение отражённого звука

Итоговая реализация использует:

RayCount = 25
